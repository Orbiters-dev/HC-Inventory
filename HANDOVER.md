# HC-Inventory 수출 비용 계산기 — 인계 문서

> 외부 거래처(단일 계정 `HC`)용 수출 비용 계산 도구.
> **이 저장소는 공개(public)** 이므로 비밀번호·토큰·서버 IP 등 민감 정보는 이 문서에 적지 않습니다 — **별도(전화/대면 등)로 전달**받으세요.

---

## 1. 사용하는 직원용 (이것만 알면 됩니다)

| 항목 | 내용 |
|---|---|
| **사이트 주소** | https://tools.orbiters.co.kr/calculator |
| **아이디** | `HC` |
| **비밀번호** | 운영자에게 별도로 전달받으세요 (첫 로그인 후 꼭 변경) |
| **비밀번호 변경** | 좌측 메뉴 → **프로필** → 비밀번호 변경 |

### 할 수 있는 일
- **수출 비용 계산기**: 제품 규격·원가·채널별(자사몰/아마존/월마트) 예상 판매가를 입력하면 미국 도착 후 손익을 계산합니다.
- **제품 정보(제품명·메모)**: 계산할 때 제품명과 메모를 달아 이력에서 구분합니다.
- **결과 복사 / CSV 내려받기**: 계산 결과 카드 우측의 `복사`·`CSV` 버튼으로 거래처에 견적을 공유합니다(엑셀에서 한글 안 깨짐).
- **계산 이력**: 좌측 메뉴 → 계산 이력. 항목을 클릭하면 그때 계산한 전체 내용(입력·결과·메모)이 펼쳐지고, **`이 이력 삭제`** 버튼으로 지울 수 있습니다.
- **프로필**: 계정 정보 확인 + 비밀번호 변경.

## 2. 관리·개발하는 직원용

### 구성 (stack)
- **백엔드**: Django 5.2 + DRF (`backend/`)
- **프론트엔드**: Next.js 16 standalone (`frontend/`)
- **DB**: PostgreSQL (이 사이트 전용, 다른 시스템과 분리)
- **배포 서버**: AWS EC2 (기존 시스템과 격리된 별도 서비스)
- **도메인**: tools.orbiters.co.kr (Gabia DNS, HTTPS 자동 갱신)

### 저장소 / CI
- repo: `github.com/Orbiters-dev/HC-Inventory`
- CI: push 시 백엔드(import 검사 + migration + check) + 프론트(lint + 빌드) 자동 검증 (`.github/workflows/ci.yml`)
- **푸시 토큰은 운영자에게 별도 요청** (`workflow` 권한 포함 필요)

### 배포
1. 코드 수정 → 커밋 → main 에 push (CI 통과 확인)
2. EC2에서 코드 받기 → 백엔드 migration/빌드 → 프론트 빌드 → 서비스 재시작
   - 정확한 명령·서버 경로·서비스 이름은 **내부 인계서**(아래) 참조

### 계정 추가 / 비밀번호 재설정
- 비밀번호 재설정: 서버에서 `manage.py changepassword HC`
- 계정 추가(관리자): 서버에서 `manage.py createsuperuser`
- 단일 계정 운영이라 권한 시스템은 없습니다.

### 데이터
- 계산할 때마다 이력(CalculationLog)이 자동 저장됩니다.
- 이력 삭제는 사이트(계산 이력 → 상세 → `이 이력 삭제`) 또는 운영자가 DB에서 처리.
